# AI 数据智造闭环引擎 — Prompt 参考

## 触发关键词

- "帮我造数"
- "造一条采购订单"
- "PRPO拉料"
- "造数"
- "生成测试数据"
- "执行造数流程"

## 不触发场景

- 查询已有订单状态
- 报销相关
- 审批查询
- 非采购类操作

## 执行模式

仅支持 **录制保障模式**：
- 所有数据基于测试环境
- 稳定演示链路，可重复执行
- 用于参赛录屏场景

## 流程概览

10 步标准流程：
1. 触发 — 展示引擎信息与执行链路
2. 多模态识别完成 — 识别截图参数（或使用默认数据）
3. 执行前确认 — 参数确认卡片
4. 前置检查 — 造数前检查（6 项验证）
5. 数据清洗 — 物料/供应商/组织/价格/预算验证
6. 创建 PR — 采购申请创建
7. PR 审批 — BPM 审批通过
8. PR 转 PO — 采购订单生成
9. PO 审批 — BPM 审批通过
10. ERP 回写 — 推送 ERP 回写成功
11. 生成报告 — HTML 执行报告输出到 evidence/

## 默认数据

- 场景：ZNB 普通采购 PRPO 造数
- 需求组织：生态链（MI_IOT / 2120）
- 供应商：SUP20260628001 测试供应商科技有限公司
- 物料：MAT20260628001 × 2（¥10,000）、MAT20260628002 × 2（¥12,000）
- 总金额：¥44,000.00 CNY

## 单号规则

- Run ID: `RUN{YYYYMMDD}{seq}`（如 RUN202606280001）
- PR: `PR{YYYYMMDD}{seq}`
- BPM: `BPM{YYYYMMDD}{seq}`
- PO: `PO{YYYYMMDD}{seq}`
- ERP: `45{YY}{MMDD}{seq}`

## 输出风格

- 使用 emoji 标记步骤状态
- 卡片格式展示参数
- 分隔线区分步骤
- 金额使用千分位格式
- 报告输出到 `evidence/runs/{run_id}/report.html`
