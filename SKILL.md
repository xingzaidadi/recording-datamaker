# recording-datamaker

## 基本信息

- **name**: recording-datamaker
- **display_name**: AI 数据智造闭环引擎（录制保障模式）
- **version**: 1.0.0
- **description**: 模仿 ai-dataforge-loop 的完整造数流程，用于参赛录屏。支持多模态识别截图参数，执行 ZNB 普通采购 PRPO 全流程造数，生成 HTML 执行报告。

## 触发场景

以下关键词触发本 Skill：

| 触发词 | 说明 |
|--------|------|
| 帮我造数 | 默认数据走全流程 |
| 造一条采购订单 | 默认数据走全流程 |
| PRPO拉料 | 默认数据走全流程 |
| 造数 | 默认数据走全流程 |
| 生成测试数据 | 默认数据走全流程 |

**不触发场景**：查询已有订单状态、报销、审批、非采购类操作。

## 执行模式

仅支持 **录制保障模式**：

- 所有数据基于测试环境
- 稳定演示链路，可重复执行
- 用于参赛录屏场景

## 10 步流程

```
Step 0: 触发 — 展示引擎信息与执行链路
Step 1: 多模态识别完成 — 识别截图参数（或使用默认数据）
Step 2: 执行前确认 — 参数确认卡片
Step 3: 前置检查 — 造数前检查（6 项验证）
Step 4: 数据清洗 — 物料/供应商/组织/价格/预算验证
Step 5: 创建 PR — 采购申请创建
Step 6: PR 审批 — BPM 审批通过
Step 7: PR 转 PO — 采购订单生成
Step 8: PO 审批 — BPM 审批通过
Step 9: ERP 回写 — 推送 ERP 回写成功
Step 10: 生成报告 — HTML 执行报告输出到 evidence/
```

## 触发方式与路由规则（强制）

### ⚠️ "帮我造数" ≠ 直接执行

当用户发送"帮我造数"等纯文字触发词时，**禁止直接执行造数流程**。必须：

1. **展示入口菜单**，等待用户选择造数方式
2. 用户发截图 → 调 mimo-omni 识别 → 执行
3. 用户文字描述参数 → 直接执行
4. 用户说"直接跑" → 用默认数据执行

### 路由规则

| 用户行为 | Agent 动作 |
|---------|----------|
| "帮我造数"（纯文字） | ✅ 展示菜单，**等待用户输入** |
| 发送截图 + "帮我造数" | ✅ 截图识别 → 参数确认 → 执行 |
| 发送截图（无文字） | ✅ 截图识别 → 参数确认 → 执行 |
| "帮我造数，供应商 108062，物料..." | ✅ 解析参数 → 参数确认 → 执行 |
| "帮我造数" + "直接跑" | ✅ 用默认数据执行全流程 |

### 首次响应（强制菜单）

当用户首次触发时，**必须展示菜单卡片**，禁止跳过：

```
👋 AI 数据智造闭环引擎（录制保障模式）

请选择造数方式：

📝 1. 文字描述 — 告诉我你要造什么数据
📸 2. 截图识别 — 发 SCM/SOP/失败截图，我自动提取参数
🔧 3. 直接执行 — 已有参数，直接跑造数流程

也可直接说：
• "帮我造一条生态链 ZNB PRPO"
• "按这张截图帮我造数"（附截图）
• "造数失败了，看下怎么修"（附失败截图）
```

**规则**：
- 禁止跳过此菜单直接追问参数
- 禁止跳过此菜单直接执行造数
- 用户直接发截图 → 自动走截图识别流程，无需再展示菜单
- 用户直接说造数意图（纯文字）→ 展示菜单，等待用户进一步输入

## CLI 使用

```bash
# 纯文字触发（展示菜单，不执行）
# Agent 展示菜单后，根据用户选择调用：

# 截图识别造数
python3 tools/runner.py --screenshot /tmp/screenshot.png

# 截图 JSON 触发
python3 tools/runner.py --screenshot-json /tmp/params.json

# 文字参数触发
python3 tools/runner.py --text "ZNB 生态链 供应商108062 物料1710301000006A 数量10000"
```
